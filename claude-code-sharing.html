// quick clarification · -p flag

What does
-p mean?

📌
-p = --print
Short for --print, not "prompt".

Runs Claude Code non-interactively — processes the query, prints output to stdout, then exits. No session opened.
🔄
vs Interactive mode
Without -p: opens a persistent REPL session.

With -p: one-shot execution. Output piped to stdout. Process exits when done.
⚡
Also known as
Previously called "headless mode" in older docs. The flag and behavior are the same — only the name changed.
💡
All CLI flags — including --append-system-prompt and --output-format — work with -p. It's the standard entry point for scripting and automation.
// tip 01 · system prompt customization

--append-system-prompt

📌
What it does
Appends your custom instructions to the end of Claude Code's default system prompt.

Does not replace — preserves all built-in capabilities.
📅
Interactive mode support
The flag existed before, but from v1.0.51 (July 8, 2025) it also works in interactive mode.

Non-interactive (-p) was supported earlier.
💡
Official recommendation
"For most use cases, use an append flag. Appending preserves Claude Code's built-in capabilities while adding your requirements."
// tip 01 · all 4 system prompt flags

4 flags,
4 use cases

FlagBehaviorMode
--system-promptReplaces the entire default system promptinteractive + non-interactive ✓
--system-prompt-fileReplaces with contents from a fileinteractive + non-interactive ✓
--append-system-promptAppends to the default prompt — recommendedinteractive + non-interactive ✓
--append-system-prompt-fileAppends from a fileinteractive + non-interactive ✓
⚠️
--system-prompt and --system-prompt-file are mutually exclusive. Append flags can be combined with either replacement flag.
📝
All 4 flags work in both interactive and non-interactive mode. Earlier docs incorrectly stated the -file variants were non-interactive only — this has since been corrected.
// tip 01 · CLAUDE.md vs --append-system-prompt

Two different
inject mechanisms

📄
CLAUDE.md
Injected as the first user message — placed after the default system prompt.

Persistent. Auto-loaded from project root or global (~/.claude/CLAUDE.md).
🔧
--append-system-prompt
Appended directly into the system prompt — not a user message.

Per-session. Must be passed on every invocation.
💡
Neither CLAUDE.md nor --append-system-prompt modifies Claude Code's default system prompt — they are two different ways to inject additional content at different positions in the context.
✅
Use CLAUDE.md for
Persistent project conventions, coding standards, team rules that apply across all sessions.
⚡
Use --append for
Task-specific behavior for a single run — CI/CD pipelines, specialized scripts, one-off instructions.
// tip 01 · examples

Real-world
examples

bash
# Add security focus for a single review run
claude -p "Review this auth code" \
  --append-system-prompt "Check for: SQL injection, XSS, insecure token handling."
bash
# Load role instructions from a file
claude -p "$(cat prompt.txt)" \
  --append-system-prompt "$(cat ROLE.md)" \
  --permission-mode acceptEdits
bash
# Interactive mode: persistent focus across the whole session
claude --append-system-prompt "Always analyze time & space complexity when writing code."

// tip 02 · how costUSD is calculated

How is
costUSD computed?

input_tokens × $5 / MTok
+ output_tokens × $25 / MTok
+ cache_creation_input_tokens × $6.25 / MTok (5‑min write)
+ cache_read_input_tokens × $0.50 / MTok
= costUSD

* Opus 4.6 pricing. Sonnet / Haiku have different rates.

Verified against the sample above (Opus 4.6):
  3       × $5 / MTok     = $0.000015   (input)
  15      × $25 / MTok    = $0.000375   (output)
  11,222 × $6.25 / MTok  = $0.070138   (cache write)
  6,285  × $0.50 / MTok  = $0.003143   (cache read)
→ Total = $0.07367 ✓ exact match with reported costUSD
// tip 02 · why does a tiny query cost so much?

Cache drives
>95% of cost

🧠
Cache write — ~$0.070
11,222 tokens = Claude Code's system prompt being cached for the first time.

This is the dominant cost — but subsequent calls will read from cache at 12.5× cheaper rates.
⚡
Cache read — ~$0.003
6,285 tokens = context reused from a previous session's cache.

Only $0.50/MTok vs $5/MTok for fresh input — 90% cheaper.
💬
Actual query — ~$0.0004
3 input + 15 output tokens = the real question and answer.

Less than 1% of total cost — the prompt itself is almost free.
💡
Takeaway: Cost is front-loaded by cache writes on the first call. Subsequent calls in the same context become progressively cheaper as more tokens shift to cache reads.
// tip 02 · is costUSD accurate?

Is
costUSD accurate?

✅
Yes — server-side
Computed by Anthropic server-side, not a client estimate.

total_cost_usd in the result message is the authoritative value for billing reconciliation.
📊
total_cost_usd vs costUSD
total_cost_usd — sum for the entire turn/session.
modelUsage[...].costUSD — per-model breakdown.

They match when only one model is used.
⚡
Multiple calls
Each claude -p invocation returns its own costUSD. The SDK does not accumulate totals — sum them yourself if you need session-level tracking.
🎯
Quick extract: claude -p "..." --output-format json | jq '.total_cost_usd'
// combining both tips

Production
script pattern

bash – CI/CD
#!/bin/bash

RESULT=$(claude -p "Review PR: $(cat diff.txt)" \
  --append-system-prompt "Focus: OWASP Top 10, auth issues, data leaks." \
  --output-format json \
  --bare)

# Extract result and cost
REVIEW=$(echo $RESULT | jq -r '.result')
COST=$(echo $RESULT | jq '.total_cost_usd')

echo "Done. Cost: \$${COST}"
echo $REVIEW
--output-format json
Parse output programmatically. Extract .result and .total_cost_usd with jq.
--bare
Skips hooks, plugins, and CLAUDE.md discovery. Recommended for CI — guarantees consistent behavior on every machine.
--append-system-prompt
Add domain-specific rules without replacing Claude Code's default capabilities.
