Multiple Accounts

CLAUDE_CONFIG_DIR — isolate work & personal profiles

1

Skill/plugin conflicts — personal vibe coding triggers company conventions, hooks blocking push to main

2

Credential conflict — ~/.claude stores shared credentials, login/logout in one session affects all others

3

Context bloat — personal skills/MCPs inflate context when using the work account

4

Login/logout hell — constantly switching between accounts wastes time

Solution: CLAUDE_CONFIG_DIR undocumented

# ~/.zshrc — alias for personal profile alias claude-personal="CLAUDE_CONFIG_DIR=~/.claude-personal claude" # First run will prompt login for personal account $ claude-personal # Configure personal profile separately $ vim ~/.claude-personal/settings.json

Before — shared ~/.claude

~/.claude/ credentials.json ← both accounts compete settings.json ← skills/hooks mixed up projects/
→

After — isolated

~/.claude/ work profile credentials.json settings.json hooks, conventions... ~/.claude-personal/ personal profile credentials.json settings.json custom skills, no hooks
