SubAgent — Comparison & Gotchas

Main Agent vs SubAgent

Main Agent SubAgent
Context Shared session (entire conversation) Fresh, isolated (separate)
Spawn agents Yes No
Tools All built-in + MCP Inherit all OR Restrict via tools/disallowedTools
MCP Servers All configured Inherit all OR Can scope via mcpServers
Skills Lazy — description only, full on invoke Must explicitly list via skills → injects full body immediately
Deferred loading ToolSearch available Not available

Gotcha #1: Eager Skill Loading

20 skills x 5K = 100K tokens consumed immediately on spawn!

Only 60K left for the task → ONLY ASSIGN NEEDED SKILLS

# .claude/agents/payroll-agent.md --- name: payroll-agent tools: Bash, Read, Edit skills: payroll-calc, salary-report # ONLY 2 skills! ---

Gotcha #2: Cannot Nest

Workarounds

1
Chain from Main
Main spawns A, gets result, then spawns B with A's output
Official
2
Use Skills instead
Skills run in main context — no nesting needed
Official
3
Bash: claude -p "task" --agent C
SubAgent runs CLI subprocess → output returns via Bash tool result. Tested: works (exit 0, output captured)
Unofficial hack
⚠ Option 3 is NOT officially supported — separate session (no shared context, extra API cost). Use at your own risk.
